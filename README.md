DATABASE TABLES (10 Tables)
═══════════════════════════════════════════════

--- TABLE 1: users ---
- id          → INTEGER, PK, AUTO INCREMENT
- name        → STRING, NOT NULL
- email       → STRING, NOT NULL, UNIQUE
- password    → STRING, NOT NULL (bcrypt hashed)
- createdAt   → TIMESTAMP (auto)
- updatedAt   → TIMESTAMP (auto)

--- TABLE 2: roles ---
- id          → INTEGER, PK, AUTO INCREMENT
- role_name   → ENUM('admin', 'librarian', 'student'), NOT NULL
- description → STRING
- timestamps: false

--- TABLE 3: role_data ---
- id          → INTEGER, PK, AUTO INCREMENT
- user_id     → INTEGER, NOT NULL, FK → users.id, onDelete CASCADE
- role_id     → INTEGER, NOT NULL, FK → roles.id, onDelete CASCADE
- assigned_at → DATE, DEFAULT NOW
- timestamps: false

--- TABLE 4: categories ---
- id          → INTEGER, PK, AUTO INCREMENT
- name        → STRING, NOT NULL, UNIQUE
- description → STRING
- createdAt, updatedAt (auto)

--- TABLE 5: books ---
- id              → INTEGER, PK, AUTO INCREMENT
- title           → STRING, NOT NULL
- author          → STRING, NOT NULL
- isbn            → STRING, UNIQUE
- category_id     → INTEGER, FK → categories.id, onDelete SET NULL
- total_copies    → INTEGER, DEFAULT 1
- avail_copies    → INTEGER, DEFAULT 1
- cover_image     → STRING (Cloudinary URL)
- cover_public_id → STRING (Cloudinary public_id for deletion)
- createdAt, updatedAt (auto)

--- TABLE 6: borrow_records ---
- id          → INTEGER, PK, AUTO INCREMENT
- student_id  → INTEGER, NOT NULL, FK → users.id, onDelete CASCADE
- book_id     → INTEGER, NOT NULL, FK → books.id, onDelete CASCADE
- borrow_date → DATE, DEFAULT NOW
- due_date    → DATE, NOT NULL
- return_date → DATE, NULL (filled on return)
- status      → ENUM('borrowed', 'returned'), DEFAULT 'borrowed'
- timestamps: false

--- TABLE 7: fines ---
- id               → INTEGER, PK, AUTO INCREMENT
- borrow_record_id → INTEGER, NOT NULL, FK → borrow_records.id, onDelete CASCADE
- student_id       → INTEGER, NOT NULL, FK → users.id, onDelete CASCADE
- amount           → FLOAT, NOT NULL (auto-calculated)
- status           → ENUM('unpaid', 'paid'), DEFAULT 'unpaid'
- paid_at          → DATE, NULL (filled when marked paid)
- createdAt, updatedAt (auto)

--- TABLE 8: reservations ---
- id          → INTEGER, PK, AUTO INCREMENT
- student_id  → INTEGER, NOT NULL, FK → users.id, onDelete CASCADE
- book_id     → INTEGER, NOT NULL, FK → books.id, onDelete CASCADE
- status      → ENUM('pending', 'fulfilled', 'cancelled'), DEFAULT 'pending'
- reserved_at → DATE, DEFAULT NOW
- expires_at  → DATE, NOT NULL (auto set: reserved_at + 7 days)
- timestamps: false
NOTE: A student can only reserve a book if avail_copies = 0.
      When a book is returned, check if any pending reservation exists,
      auto-notify that student via email that book is now available.
      One student cannot reserve the same book twice with status 'pending'.

--- TABLE 9: reviews ---
- id         → INTEGER, PK, AUTO INCREMENT
- student_id → INTEGER, NOT NULL, FK → users.id, onDelete CASCADE
- book_id    → INTEGER, NOT NULL, FK → books.id, onDelete CASCADE
- rating     → INTEGER, NOT NULL (1 to 5 only, validate range)
- comment    → TEXT, NULL
- createdAt, updatedAt (auto)
NOTE: A student can only review a book they have borrowed (check borrow_records).
      One student can only submit one review per book (unique constraint on student_id + book_id).

--- TABLE 10: otp_tokens ---
- id         → INTEGER, PK, AUTO INCREMENT
- user_id    → INTEGER, NOT NULL, FK → users.id, onDelete CASCADE
- otp        → STRING, NOT NULL (6-digit numeric OTP)
- expires_at → DATE, NOT NULL (NOW + OTP_EXPIRE_MINUTES from .env)
- is_used    → BOOLEAN, DEFAULT false
- createdAt  → TIMESTAMP (auto)
- timestamps: false

═══════════════════════════════════════════════
MIDDLEWARE
═══════════════════════════════════════════════

auth.js:
- Extract Bearer token from Authorization header
- Verify JWT using JWT_SECRET
- Attach decoded user_id to req.user
- If invalid or missing → 401 Unauthorized

rbac.js:
- Accept allowedRoles array e.g. ['admin', 'librarian']
- Query role_data WHERE user_id = req.user.id
- JOIN roles table to get role_name
- If role_name in allowedRoles → next()
- Else → 403 Forbidden

Middleware flow:
Request → auth.js → rbac.js → Controller

FORGOT PASSWORD VIA OTP (utils/otpGenerator.js)
═══════════════════════════════════════════════
Flow:
Step 1 — POST /api/auth/forgot-password
- Accept email in request body
- Find user by email, if not found → 404
- Generate 6-digit numeric OTP using crypto.randomInt(100000, 999999)
- Save OTP in otp_tokens table with expires_at = NOW + OTP_EXPIRE_MINUTES
- Invalidate all previous unused OTPs for same user (set is_used = true)
- Send OTP to user email via Nodemailer
- Response: { message: 'OTP sent to your email' }

Step 2 — POST /api/auth/verify-otp
- Accept email + otp in request body
- Find OTP in otp_tokens WHERE user_id matches, is_used = false, expires_at > NOW
- If not found or expired → 400 Bad Request
- Mark OTP as is_used = true
- Generate a short-lived reset JWT (expires in 15 minutes) with user_id
- Response: { reset_token: '...' }

Step 3 — POST /api/auth/reset-password
- Accept reset_token + new_password in request body
- Verify reset_token using JWT_SECRET
- Hash new password with bcrypt
- Update user password in users table
- Send confirmation email via Nodemailer
- Response: { message: 'Password reset successful' }

RESERVATION LOGIC
═══════════════════════════════════════════════
- Student can only reserve a book if avail_copies = 0
- If avail_copies > 0, return error: 'Book is available, please borrow directly'
- One student cannot have two pending reservations for the same book
- expires_at = reserved_at + 7 days (auto set on creation)
- On book return → query reservations WHERE book_id = ? AND status = 'pending' ORDER BY reserved_at ASC LIMIT 1
- If pending reservation found → send email to that student: 'Your reserved book is now available'
- Librarian/admin can fulfil or cancel any reservation
- Student can only cancel their own pending reservation

═══════════════════════════════════════════════
REVIEW LOGIC
═══════════════════════════════════════════════
- Student can only review a book if borrow_records contains a record with their student_id + book_id
- rating must be INTEGER between 1 and 5 (validate strictly)
- Unique constraint on (student_id + book_id) → one review per book per student
- GET /api/books/:id → include average rating calculated from reviews
- GET /api/reviews/book/:book_id → return all reviews with student name and rating

CATEGORY LOGIC

- Admin/Librarian can add, edit categories

FINE CALCULATOR (utils/fineCalculator.js)
═══════════════════════════════════════════════
- Fine = days_late × FINE_PER_DAY (from .env, default ₹5)
- days_late = return_date - due_date (in days, only if positive)
- If return_date <= due_date → no fine generated
- Fine auto-created on return if days_late > 0
- Fine status starts as 'unpaid'
- Only admin/librarian can mark as 'paid' via API
